# 功能介绍报告 — LLM 时尚推荐系统

> 本文档介绍系统的整体定位、核心功能、技术架构与数据流,帮助你快速理解「这个项目能做什么、是怎么做的」。

---

## 一、项目定位

这是一个面向**时尚电商**场景的 **AI 导购系统**。它把「检索增强生成(RAG)」「多轮对话」「虚拟试穿」「图生 3D」「WebXR 沉浸式展示」串成一条完整链路,模拟用户从「描述需求 → 得到推荐 → 试穿 → 3D/VR 查看」的购物体验。

一句话:**用大模型当导购,用向量检索找货,用图像/3D 生成让用户「看见自己穿上」。**

---

## 二、核心功能

### 1. AI 时尚推荐(文本 → 商品)
用户用自然语言描述需求(如 "a red summer dress under 1000"),系统返回最匹配的商品,并用 LLM 生成自然语言推荐理由。

### 2. 多轮对话理解
支持上下文追问。用户先说 "推荐连衣裙",再说 "要红色的"、"便宜点的",系统会自动把追问改写成完整查询(query rewrite),理解承接上文的意图。

### 3. 混合检索(Hybrid Retrieval)
不依赖单一检索方式,而是融合多种召回 + 重排,显著提升准确率:
- **FAISS**:向量语义检索
- **BM25**:关键词检索(兜底召回)
- **ChromaDB + Self-Query**:LLM 把自然语言转成结构化元数据过滤条件(如价格、颜色)
- **Cross-Encoder**:对候选结果做精排重排序

### 4. 话题守卫(Topic Guard)
对偏离时尚购物的提问做拦截,保证系统聚焦在导购场景。

### 5. 虚拟试穿(图 → 图)
上传人物照片 + 选中的商品图,通过图像生成模型(Gemini-2.5-flash-image)合成「穿上这件衣服」的效果图。

### 6. 图生 3D(图 → GLB 模型)
把试穿结果图通过 Tripo3D 转成 GLB 三维模型,可用于 WebXR 3D 场景中查看。

### 7. WebXR 3D 服装店
一个基于 Three.js 的网页端 3D 服装店场景(`ClothesShop_optimized.glb`),支持第一人称浏览,可作为沉浸式展示前端。

## 五、推荐工作流(LangGraph)

推荐请求 `/recommend` 在后端走的是一条由 LangGraph 编排的多节点流水线:

```
用户查询
   │
   ▼
① query_rewrite      ── 结合对话历史,把追问改写成完整查询(支持多轮)
   │
   ▼
② check_topic        ── 话题守卫,过滤与时尚购物无关的问题
   │
   ▼
③ self_query_retrieve ── ChromaDB,用 LLM 生成的元数据过滤条件检索
   │
   ▼
④ ranker (fallback)  ── FAISS + BM25 召回,再用 Cross-Encoder 重排
   │
   ▼
⑤ rag_recommender    ── 综合检索结果,生成最终自然语言推荐
   │
   ▼
返回推荐商品 + 推荐理由
```

每个节点职责单一、可独立替换,这也是用 LangGraph 而非单一 prompt 的原因——流程可观测、可调试、可扩展。

## 六、主要 API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/recommend/` | 多轮时尚推荐(RAG 主链路) |
| `POST` | `/try-on` | 虚拟试穿(人物照片 + 商品图) |
| `POST` | `/img2model` | 图生 3D GLB 模型(Tripo3D) |
| `GET` | `/trending` | 热门 / 趋势商品 |
| `GET` | `/download/{filename}` | 下载生成的 GLB(供 WebXR 加载) |
| `GET` | `/health` | 健康检查 |
| `GET` | `/docs` | Swagger 交互式 API 文档 |

启动后端后,访问 http://localhost:8000/docs 可在线试调所有接口。

## 八、数据与索引

- **数据集**:Farfetch.com 时尚商品数据(`backend/data/FashionDataset.csv` → 清洗为 `processed_data.csv`)。
- **索引(已随包提供,开箱即用)**:
  - FAISS 向量索引
  - BM25 关键词索引
  - ChromaDB(自查询,含元数据)
  - Cross-Encoder 重排序模型
- 如需从原始数据重建索引:
  ```powershell
  $env:PYTHONPATH = "backend"
  uv run python -m src.indexing.embedding
  ```
  会重新生成 `backend/data/indexes/` 下的全部文件。

## 九、设计亮点小结

1. **多路召回 + 精排**:FAISS(语义)+ BM25(关键词)+ Self-Query(结构化过滤)+ Cross-Encoder(重排),互补提升准确率与鲁棒性。
2. **真·多轮**:用 query rewrite 把口语化追问还原为完整意图,而非简单拼接历史。
3. **模块化流程**:LangGraph 把推荐拆成可观测、可替换的节点,便于调试与扩展。
4. **端到端体验闭环**:文本推荐 → 虚拟试穿 → 图生 3D → WebXR 展示,覆盖完整购物想象链路。
5. **本地嵌入 + 云端生成**:嵌入/重排在本地跑(省钱、低延迟),只有 LLM 对话和图像/3D 生成走云端 API。
